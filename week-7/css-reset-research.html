CSS Reset Research

Historical Foundation - Eric Meyer, 2007

What browser inconsistencies bothered developers enough to create this?

Because Internet Explorer does not support inherit, a reset was created in order to get all browsers on the same page and create consistencies between the same web pages or sites on different browsers. The reset allows everything to be set specifically and not relying on the browsers default settings.

Modern Evolution - Josh Comeau, 2025

Why does Josh modify or reject parts of Meyer's approach?

Josh keeps and works with the browsers default settings but makes sure to add his own to make it more accessible, pretty, and easier to work with. He uses inherit in the same way by applying it to font but he also has multiple other additions that make it more modern and apply it to modern web browsers such as antialiased, overflow wrap, and using images and video among other assets.

Critical Questions

Why reset at all? (What problem does it solve?)

A CSS reset is used because every browser has its own default styles and those differences can create inconsistencies across everything. By resetting everything, designers can build from the same universal foundation instead of having to individually fight whichever browser's quirks. It solves the problems of uneven margins, heading differences, and other weird styling defaults.

Why Josh's version over Eric's? (Be specific about 2–3 differences)

Josh’s version improves on Eric's version by being more modern and intentional. Eric stripped everything down to zero so developers had to rebuild everything from scratch while Josh’s reset is softer and keeps some of the useful things like system fonts while simultaneously normalizing the rest. He also adds small improvements like box sizing, font rendering, and better accessibility.

How do you actually use these? (Link tag? Import? Inline?)

Using the reset is easy since it is linked in the HTML like any stylesheet. Link the reset.css file in the head of the document.

What's a CSS boilerplate? (vs. a reset)

A boilerplate gives the starting structure that gives common elements like typography, buttons, and layout while a reset you have to build up a lot more.

Why use a boilerplate? (When does it help/hurt?)

A boilerplate would help when a file needs to be developed quickly with consistency across pages which saves time. It can hurt if you don’t understand what’s in there and can’t change it to fit the design.

Why is Normalize.css different from resets? (Philosophy shift)

Normalize.css is different from resets because it doesn’t try to erase browser styles and instead works with them. It makes small adjustments to bring all the browsers' different styles of understanding closer together to make things more cohesive and keep their useful defaults intact. The philosophy shift focuses on refinement rather than erasing everything and starting at zero. Normalize keeps the good and alters the bad which creates smoother design systems without starting from scratch.
